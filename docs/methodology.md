# Methodology

## Overview

Zero Trust Adapt uses a structured methodology to translate survey responses into a clear and actionable security roadmap.

The goal is not only to assess the current state, but to determine:
- What is already implemented and can be improved
- What is missing and needs to be introduced
- What should be prioritised based on risk and business context

---

## Assessment Structure

The assessment is divided into key security domains:

- Users
- Endpoints & Devices
- Internet Traffic
- Networks
- Applications
- Data Loss & Logging
- Automation & Steady State

Each domain contains multiple-choice questions designed to represent different levels of security maturity.

Answers are structured from:
- Low security / traditional approach  
to  
- High security / Zero Trust aligned approach

---

## Evaluation Logic

Each response is analysed using three criteria:

### 1. Current State
Identifies what controls or practices are already in place.

### 2. Gap Identification
Determines what is missing compared to a Zero Trust approach.

### 3. Improvement Potential
Highlights controls that exist but require optimisation rather than replacement.

---

## Maturity Classification

Each domain is evaluated and classified into a maturity level:

- Basic
- Developing
- Intermediate
- Advanced
- Zero Trust Aligned

This allows the report to provide context-aware recommendations rather than generic advice.

---

## Report Generation

Based on the combination of responses, the system generates a tailored report structured into:

### 1. Executive Summary
High-level overview of the organisation’s security posture.

### 2. Key Findings
Main gaps and strengths identified across all domains.

### 3. Domain Analysis
Detailed breakdown per domain, including:
- Current state
- Identified gaps
- Recommended improvements

### 4. Action Plan
A prioritised roadmap divided into:
- Short-term actions (quick wins)
- Mid-term improvements
- Long-term strategic changes

---

## Recommendations Approach

Recommendations are designed to be:

- Practical and realistic
- Adapted to company size and maturity
- Aligned with modern cybersecurity principles

Where relevant, suggestions include:
- Software solutions (e.g. MFA, EDR, SIEM)
- Hardware considerations
- Process and policy improvements

---

## Objective of the Methodology

The purpose of this methodology is to simplify decision-making.

Instead of overwhelming organisations with complex frameworks, the system provides:
- Clarity
- Structure
- Actionable steps

This allows organisations to move progressively towards a Zero Trust architecture in a controlled and realistic way.
